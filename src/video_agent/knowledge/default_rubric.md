# Demo video evaluation (observe only what is visible or audible)

Score each row: **0–5** (integer).

- **5**: Exceeds expectations
- **4**: Meets expectations
- **2**: Partial evidence / needs improvement
- **1**: Missing / weak evidence
- **0**: Not observable from the recording (say what’s missing in evidence)

## A. Submission and format
- **A1** Length/format acceptable; file plays; resolution viewable.
- **A2** Project and team identified early (if required by course policy).
- **A3** Clear narration or readable on-screen text for key steps.

## B. Structure and communication
- **B1** Logical flow (context → demo → wrap-up).
- **B2** Explains actions and ties them to project goals.
- **B3** Pacing appropriate; critical screens visible long enough.

## C. Demonstration depth
- **C1** Primary user journey shown end-to-end.
- **C2** Required features from the assignment list are shown in action, or clearly marked not implemented.
- **C3** If integrations are required, they are shown in use (not only named).
- **C4** Error or edge path shown (only if the assignment requires it).

## D. Product and UX (visible only)
- **D1** UI legible; important states visible.
- **D2** Navigation and main actions understandable.
- **D3** No obvious broken main flow unless discussed.

## E. Integrity
- **E1** Claims match what is shown.
- **E2** Limitations, mocks, or unfinished areas disclosed when applicable.
- **E3** Mentions alignment with repo/commit if that is claimed on video.

## F. Optional
- **F1** Setup or reproducibility cues if required.
- **F2** Polish/accessibility only if clearly visible.

When scoring **0**, explain what was missing from the recording.
